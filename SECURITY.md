# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability within this repository, please do **NOT** open a public issue.

Instead, please report it directly via GitHub Private Vulnerability Reporting or contact Kris Vandebroek.

### What to include in your report:

- A description of the vulnerability and its potential impact.
- Step-by-step instructions or proof-of-concept to reproduce the issue.
- Any suggested mitigations or fixes.

We take security seriously and will review and respond to reports promptly.
